# Extra Molecules Pickle Support - Design Decision

## Status: POSTPONED (Requires Python Preprocessing Path)

## Overview

Extra molecules in Boltz are non-standard residues and ligands that are loaded from pickle files containing RDKit `Mol` objects.

## Python Implementation

### File Format
- **Location:** `{extra_mols_dir}/{record.id}.pkl`
- **Content:** `dict[str, Mol]` where:
  - Keys: Residue names (e.g., "ALA", "GLY", "LIG")
  - Values: RDKit `Mol` objects with full molecular structures

### Loading Code
```python
# inferencev2.py, lines 94-100
extra_mols = {}
if extra_mols_dir is not None:
    extra_mol_path = extra_mols_dir / f"{record.id}.pkl"
    if extra_mol_path.exists():
        with extra_mol_path.open("rb") as f:
            extra_mols = pickle.load(f)  # noqa: S301
```

### Usage in Featurizer

```python
# featurizerv2.py, lines 250-254
molecules = {}
molecules.update(self.canonicals)
molecules.update(input_data.extra_mols)
mol_names = set(tokenized.tokens["res_name"].tolist())
mol_names = mol_names - set(molecules.keys())
molecules.update(load_molecules(self.mol_dir, mol_names))

# Later in process_atom_features
mol = molecules[token["res_name"]]
conformer = mol.GetConformer(conf_id)
atoms = mol.GetAtoms()
bonds = mol.GetBonds()
```

### RDKit Dependencies

The featurizer relies heavily on RDKit APIs:
- `mol.GetAtoms()` - Iterate over atoms
- `mol.GetBonds()` - Iterate over bonds  
- `mol.GetConformers()` - Get conformer ensemble
- `mol.GetConformer(id)` - Get specific conformer coordinates
- `mol.GetProp("name")` - Get atom properties
- `random.choice()` - Random conformer sampling

## Rust Implementation Challenges

### 1. No RDKit Bindings
Rust has no native RDKit bindings. Options:
- **rdkit-rs** - Unmaintained, incomplete bindings
- **Python Bridge** - Use PyO3 to call Python from Rust (complex, slow)
- **Manual Parsing** - Reverse-engineer RDKit Mol format (extremely complex)

### 2. Complex Pickle Format
RDKit Mol objects are Python objects with:
- Internal C++ memory (RDKit is C++ library)
- Complex serialization format
- Atom objects with property dictionaries
- Bond objects with aromaticity flags, etc.

### 3. Deep Featurizer Integration

The featurizer uses RDKit extensively:
- Atom/bond iteration (hundreds of queries)
- Property extraction (hundreds of calls)
- Conformer sampling with random state
- 3D coordinate access

### 4. Maintenance Burden

Even if implemented today, would require:
- Ongoing maintenance for RDKit updates
- Python bridge compatibility issues
- Complex debugging across language boundaries

## Use Case Analysis

### When Are Extra Mols Needed?

1. **Non-standard amino acids** (rare modifications)
2. **Ligand molecules** (from CCD or custom SMILES)
3. **Modified nucleic acid residues**
4. **Post-translational modifications**

### Frequency in Boltz

In the reference codebase, extra_mols are:
- **Optional** (can be None)
- **Target-specific** (loaded only for specific records)
- **Not required** for standard protein inference
- **Preprocessed** (expected to be pre-generated by Python Boltz)

## Recommended Approach

### For Now: Keep "Not Implemented"

**Rationale:**
- Limited use case (optional, target-specific)
- High implementation cost vs. benefit
- Standard residues work without extra_mols
- Clear error message guides users

**Implementation:**
```rust
pub fn load_input(
    ...
    extra_mols_dir: Option<&Path>,
    ...
) -> Result<Boltz2InferenceInput> {
    if extra_mols_dir.is_some() {
        bail!(
            "load_input: extra_mols pickle loading is not implemented; pass extra_mols_dir=None. \
            Extra molecules require Python RDKit preprocessing (see docs/EXTRA_MOLS_PICKLE.md)."
        );
    }
    ...
}
```

### Future Options (If Use Case Emerges)

1. **Simplified JSON Format** (Recommended)
   - Pre-compute conformers as JSON/NPZ
   - Store atom/bond tables
   - Store molecular properties
   - **Pro:** No RDKit dependency, fast loading, simple Rust code
   - **Con:** Requires preprocessing update in Python Boltz

2. **RDKit FFI** (If Needed)
   - Use PyO3 to call Python from Rust
   - **Pro:** Full RDKit access, maintain Python path
   - **Con:** Slow, complex, dependency on Python environment

3. **rdkit-rs Bindings** (High Risk)
   - Bind to unmaintained library
   - **Pro:** Native Rust access
   - **Con:** Unmaintained, incomplete, compatibility issues

## Recommended Workflow

For users who need extra molecules:

1. **Preprocess with Python Boltz:**
   ```bash
   # Run Python Boltz preprocessing to generate NPZ + extra_mols pickle
   python -m boltz.main predict input.yaml --output ./preprocessed
   ```

2. **Convert to Simplified Format** (If needed):
   - Write Python script to convert RDKit Mols → JSON/NPZ
   - Store atom/bond tables
   - Store conformer coordinates
   - Store molecular properties

3. **Use Simplified Format in Boltr:**
   ```rust
   // Future: load_extra_mols_from_json()
   ```

**Current Status:** POSTPONED
